Todo

Any items listed here are available for developers to work on. If you want to work on a particular item, or have suggestions for other items to be added, please let the mailing list know.

Note that these are ordered roughly by priority, with immediate development tasks at the start and more general wish-list items towards the end.

  • Anything not yet complete on the status page.
  • Test all functions on the 'Testing' list.
  • Implement remaining kQ3XMethodTypeCameraFrustumMatrix camera method for view plane cameras.
  • Glue functions should make use of the Error Manager to report errors, and should include additional tests in debug builds.
  • Make all example apps cross platform.
  • MPW Makefiles to do builds with MrC: may provide better performance than CodeWarrior on Mac OS.
  • Be OpenGL support.
  • Matrix camera object, allows direct specification of world->screen matrix.
  • Accessors to be able to retriever various callbacks after setting them: error handlers, view progress functions, etc.
  • Publish plug-in geometry API (currently used internally, but not documented).
  • API to expose the cached form of high level geometries (i.e., access the TriMesh generated for them during rendering).
  • Error Manager, be able to turn a TQ3Error/TQ3Warning/TQ3Notice into a string suitable for displaying in a dialog.
  • QuesaLight.h, An 'eye' light type.
  • QuesaLight.h, Be able to put lights in non-light groups.
  • QuesaMath.h, #define to enable inline versions of simple routines.
  • QuesaShader.h, Q3ColorARGB_Luminance, to take a TQ3ColorARGB.
  • QuesaStyle.h, A 'cast shadow' style to complement the 'receive shadow' style.
  • QuesaStyle.h, Be able to set fill style independently for backfacing and frontfacing polygons.
  • QuesaStyle.h, A 'kQ3AntiAliasModeMaskPoints' style as per OpenGL.
  • QuesaView.h, Q3View_GetFogStyleState to access fog style state.
  • QuesaView.h, Q3View_GetAllowAllGroupCulling to access group culling state.
  • QuesaStyle.h, subdivision style that allows use of c2 in world and screen space subdivision, allows things like torus to control subdivision in both u and v.
  • Behaviours, to be able to associate some sort of behaviour with a TQ3Object, and for that behaviour to be invoked automatically during rendering.
  • File importers. 3DMF and 3DS are being worked on, DXF and VRML have been proposed.
  • API extension, generate a group of normal indicators for a geometry (currently implemented internally, but not published).
  • Greater renderer control, through pre and post render callbacks called immediately before objects are passed to a renderer (e.g., to enable API specific functionality like disabling a Z-buffer on an API like OpenGL).
  • 3D text geometry object, takes text strings and creates 3D representations of them.
  • General extrusion library, takes a TQ3Bitmap and creates a 3D extrusion of it.
  • AltiVec support for Mac OS (e.g., in E3Math.c).
  • Multithreading support.
  • Additional plug-in renderers (e.g., Direct3D).
  • Non-Apple documentation.
  • Customisable object registration - clients using static libraries should be able to only register the objects they need.
  • Grayscale texture support - define a kQ3PixelTypeLuminance8 pixel type, and update the OpenGL renderer to support that pixel format.
  • Be able to register a callback for when an object is disposed of: would allow the Quesa IR to free up textures automatically, rather than having to poll them on each frame.
  • IRUpdate.c, QuickTime compressed textures need to be uncompressed for passing to OpenGL.
  • A Q3Perf application for performance testing, similar to GLPerf.
  • Qut, apps should limit their frame rate when not in Show FPS mode.
  • Error manager only needs to be built in debug builds: should be a no-op in release builds.
  • Q3Xxxxx post calls should only be available to extensions.
  • Need Transform Test app to test transform objects.
  • Viewer, update about box.
  • Viewer, need Windows, Unix, Be projects.
  • Viewer, put tools in separate source files?
  • Viewer, put platform specific code in separate source files.
  • 3D World, expects to be able to call with bad params, should glue check for correct object. types even in release build and return an error?
  • Break out 3DMF importer/exporter into a plug-in, as per renderers.
  • Debugging, when dispose a shared object put it on a freed-list to catch double-disposes.
  • Performance, should we automatically add face/vertex normals to retained mode objects?
  • Qut, implement select picture/metafile for Unix/Windows.
  • Qut, implement texture importers for Unix/Windows.
  • Qut, implement mouse events for Unix.
  • TriGrid, cached form should be a TriMesh rather than a group of triangles.
  • General Polygon, need to support holes.
  • Performance, check that geometries add face/vertex normals for fastest path through OpenGL.
  • Performance, add an attribute that indicates a geometry is closed (e.g., box, ellipsoid, etc). IR can then turn on backface culling for these objects if they're not rendering in edge mode and if no transparency is active.
  • IR, doesn't support specular colour/control at the vertex level.
  • IR, doesn't support ambient coefficient.
  • IR, doesn't support transparency.
  • IR, doesn't support alpha in textures.
  • IR, add support for kQ3XMethodTypeRendererIsBoundingBoxVisible method (and call it from Quesa).
  • IR, if object is entirely visible set OpenGL frustum clip hint to turn off scissoring.
  • IR, faster to pass colour data as 3 bytes rather than 3 floats?
  • Write and execute script to generate full parameter testing and error checking for glue Q3 functions.
